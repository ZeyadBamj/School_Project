# 🎓 School Management System

## 🧠 فكرة المشروع

هذا المشروع هو نظام بسيط لإدارة مدرسة، يركز بشكل أساسي على إدارة الطلاب وربطهم بالدروس.

الفكرة بدأت كمشروع تدريبي لفهم المسارات الاساسية في البرمجة:

ومع الوقت، المشروع تطور من مجرد سكربت بسيط إلى نظام منظم فيه طبقات واضحة

## 🧩 كيف يعمل النظام؟

### 1. قاعدة البيانات (Database Layer)

تم بناء كلاس `DataBase` لإدارة الاتصال بـ SQLite.

أهم نقطة هنا:

```python
self.connection.row_factory = sqlite3.Row
```

وهذا يسمح لك بالتعامل مع النتائج كـ dictionary بدل tuple.

---

### 2. ORM (Model Layer)

تم بناء ORM بسيط جدًا يدويًا باستخدام:

- `Model` → الكلاس الأساسي
- `Field` → لتعريف أنواع الحقول

مثال:

```python
class Students(Model):
    student_id = IntegerField(unique=True, nullable=False)
    first_name = CharField(max_length=30)
```

💡 الفكرة:
بدل ما تكتب SQL يدويًاالنظام يولد الجداول تلقائيًا.

---

### 3. Repository Pattern

نمط تصميم هيكلي يُستخدم بشكل أساسي لعزل منطق التعامل مع البيانات (قاعدة البيانات) عن باقي أجزاء التطبيق.

طبقة وسيطة بدل ما تتعامل مباشرة مع قاعدة البيانات

Repository طبقة فوق طبقةال ORM
ينظم استخدامه بدل ما تستخدم ORM مباشرة في كل مكان

#### StudentRepository

- add
- update
- delete
- exists

#### LessonRepository

- ربط الطلاب بالدروس
- إدارة العلاقات (Many-to-Many)

الفائدة:

- فصل المسؤوليات
- سهولة التغيير
- سهولة الاختبار
- تنظيم الكود

---

### 4. الاختبارات (Testing)

تم استخدام:

```
unittest
```

مع قاعدة بيانات:

```
":memory:"
```

هذه نقطة مهمة جدًا:
قاعدة البيانات في الذاكرة تحل مشاكل:

- تضارب البيانات
- بطء الاختبارات
- التلوث بين الاختبارات

وتم انشاء الاختبارات للدوال التي تم انشائها فيما بعد في ملف student_repo.py و lesson_repo.py
وذلك باستخدام مبدأ TDD (Test Driven Develpoment)

StudentService

## ⚠️ :بعض المشاكل التي واجهتني

### 1. مشكلة اختفاء البيانات في الاختبارات

عند استخدام `:memory:`:

- البيانات تختفي بعد انتهاء الاتصال

✔️ الحل:

- التأكد من بقاء الاتصال مفتوح أثناء الاختبار

---

### 2. مشكلة إغلاق الاتصال (close)

كان يتم إغلاق الاتصال داخل الدوال، مما سبب:

- فشل الاختبارات
- فقدان البيانات

✔️ الحل:

- إزالة `close()` من داخل الدوال
- ترك إدارة الاتصال لمستوى أعلى

---

### 3. التحقق من المدخلات

في البداية:

- المستخدم يمكنه إدخال أي شيء

```python

```

للتأكد من:

- الأرقام
- النصوص
- القيم الفارغة
